# Disciplina Projeto de Software 2 90h (120Aulas)

## Módulo 1 - Revisão sobre Levantamentode requisitos e Metodologias Ágeis

- O papel do Analista de requisitos (PO)
    - Intermediar "Comunicação" entre StakeHolders e a equipe de Desenvolvimentos, gerenciando o Backlog do projeto

- Metodologias de desenvolvimento de projetos
    - Clássica / Cascata (Waterfall) 
        - Foco no Projeto
        - Uma etapa após a outra
    - Metodologias Ágeis (Scrum, Kanbam, XP, Lean)
        - 4 Pilares do desenvolvimento:
            - Foco no usuário (Indivíduos)
            - Entregas Rápidas (Software em funcionamento vale mais que Documentação)
            - Mudanças são bem Vindas
            - Indivíduos e Interações mais que Processos e Ferramentas

### Módulo 2 - Levantamento de Requisitos

- O que é Requisito
    -Requisitos Funcionais e Não funcionais

    -Requisito, regra do negócio e Restrição

- Documentação de Requisito

| Campo             | Conteúdo                                   | Exemplo                                                        
|-------------------|--------------------------------------------|----------------------------------------------------------------|
| ID                | Código único                               | RF-001                                                         |
| Nome              | Verbo no infinitivo + objetivo             | Cadastro de usuários                                           |
| Descrição         | O sistema deve ser capaz de…               | O cliente deve ser capaz de realizar seu cadastro na plataforma|
| Prioridade        | Importância para o negócio                 | Alta (Precisa ter)                                             |
| Regras vinculadas | Referência à RNF                           | RNF - Banco de Dados                                           |


## Módulo 3 - Técnicas de Levantamentos de Requisito

- Briefing:
    - Coleta de Dados Inicial do Projeto
    - Não Existe um Formato Único:
        - Relatório
        - Reunião
        - Telefonema, 
        - Email

- BrainStorm
    - Reunião Rápida entre uma Equipe Multidisciplinar com Objetivo de Resolver um ou mais Problemas
    - Pode acontecer de Forma Estruturada ou Não

- Questionário
    - Perguntas Direcionadas com Objetivo de Obter Informações Relevantes para Montagem do Escopo do Projeto

- Entrevistas
    - Coleta de Dados com os StakeHoldes (Pessoas Interessadas no Projeto)

- Etnografia
    - Ato de Observar o Dia-a-Dia dos Envolvidos no Projeto (CLientes e Usuários)
    - Entender a Cultura de Empresa

- Workshop
    - Apresentação do Projeto em Feiras, Eventos e Workshops com Inturio de Apresentar informações sobre o Produto desenvolvido
    - necessário de um MVP para apresentação

- Prototipagem
    - Criação de modelos visuais para melhor identificação dos elementos do Projeto
    - os Protótipos podem ser de Baixa (Rafe), Média, Alta Fidelidade
